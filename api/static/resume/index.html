System Design

Architecture snapshot of LinkVault's API, cache, queue, worker, and observability stack.

What

  • FastAPI service to create, redirect, inspect, and deactivate short links.
  • Key endpoints: POST /links, GET /{shortcode}, GET /links/{shortcode}/stats, DELETE /links/{shortcode}.
  • Health and platform visibility via /health, /metrics, and OpenAPI docs on /docs.

How

  • Postgres stores links and click counts as system-of-record data.
  • Redis caches redirect data and reduces DB latency on hot shortcodes.
  • API publishes click events asynchronously to RabbitMQ for non-blocking redirects.
  • Worker consumes events, batches updates, and flushes aggregated counts to Postgres.

Why

  • Keep redirect path fast and resilient by decoupling analytics updates from user response time.
  • Improve scalability under spikes through cache-first reads and queue-backed write smoothing.
  • Support production debugging with structured logs, distributed tracing, and metrics dashboards.
